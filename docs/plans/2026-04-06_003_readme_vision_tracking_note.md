# Plan

## Goal

비전 시스템이 운영 센서가 아니라 초기 학습 및 검증 단계의 외부 측정계라는 점만 간단히 드러내고, 기존 README의 메인 시스템 설명은 유지한다.

## Files
- `docs/plans/2026-04-06_003_readme_vision_tracking_note.md`
- `README.md`
- `docs/daily_notes/2026-04-06.md`

## Changes
- `README.md`의 가상센싱 또는 검증 단계 설명 근처에 외부 비전 기반 `XY ground-truth` 측정계 관련 문장을 최소 범위로 추가한다.
- 비전 시스템이 초기 학습 및 검증 단계에서만 사용되고, 운영 단계에서는 제거된다는 점을 명시한다.
- 운영 단계의 보정 방식이 모터 각도 실측값과 시뮬레이션 데이터를 활용한 가상센싱 기준임을 간단히 적는다.
- 변경 기록을 `docs/daily_notes/2026-04-06.md`에 추가한다.

## Impact
- README 수준에서 비전 시스템의 역할이 오해 없이 드러난다.
- 메인 시스템 구조와 데이터 계약은 변경되지 않는다.
- 상세 설계는 `docs/vision_tracking.md`에 남기고, README는 요약 수준을 유지한다.

## Risk
- 문구가 과하면 README가 운영 시스템 설명보다 실험 장치 설명에 치우칠 수 있다.
- 위치가 부적절하면 비전 시스템이 상시 운영 구성요소처럼 읽힐 수 있다.

## Validation
- README 문구가 `학습/검증용 외부 측정계`라는 성격을 명확히 드러내는지 확인한다.
- 운영 단계에서는 비전 시스템을 제거한다는 점이 분명한지 확인한다.
- README의 기존 프로젝트 목표 및 시스템 흐름과 충돌 없이 읽히는지 검토한다.
